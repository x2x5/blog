去年冬天整理书架的时候翻出一本旧笔记本，封面已经卷边了。扉页上写着一行字：

> 持续写作，持续思考 —— 不管有没有人看。

那是大学时候的自己留给现在的话。字迹有点稚嫩，但意思很清楚。那会儿我在几个平台都写过博客，后来工作忙了，域名过期了，文章散落在各处，就不再写了。今年下定决心重新开始，自己搭一个完全可控的地方。

## 技术选型

现在建博客的选择很多。我试过几个方案，最终选了 Docusaurus：

| 方案 | 类型 | 写作体验 | 定制灵活度 | 维护成本 |
|------|------|----------|------------|----------|
| Docusaurus | SSG (React) | MDX，组件可嵌入 | 高 | 低 |
| Hugo | SSG (Go) | Markdown，模板语法 | 中 | 低 |
| Next.js + MDX | SSR/SSG | 全自定义 | 最高 | 高 |
| WordPress | 动态 | 可视化编辑 | 中 | 高 |

Docusaurus 的 MDX 支持让我可以在文章里直接使用 React 组件，比如评论系统、交互式图表，这对以后写技术文章很方便。而且它内置了 i18n、搜索、RSS，开箱即用。

最终我定下的核心配置大概是这样的：

```ts
// docusaurus.config.ts
const config: Config = {
  url: 'https://x2x5.top',
  baseUrl: '/blog/',
  i18n: {
    defaultLocale: 'zh-CN',
    locales: ['zh-CN', 'en'],
  },
  presets: [
    [
      'classic',
      {
        docs: false,
        blog: {
          routeBasePath: '/',
          remarkPlugins: [remarkMath],
          rehypePlugins: [rehypeKatex],
        },
      },
    ],
  ],
};
```

关键是把 `routeBasePath` 设为 `'/'`，这样博客首页就是 `https://x2x5.top/blog/`，而不是 `blog/blog/`。中文和英文文章各自放在不同目录，Docusaurus 会自动匹配。

## 写作工作流

以前写博客最大的障碍就是想写的时候找不到状态。这次我给自己定了几条规则：

1. **想到就记** —— 手机备忘录先写标题和两三句话，不追求完整
2. **每周整理** —— 把碎片扩展成段落，放进 `blog/YYYY-MM-DD-slug/index.mdx`
3. **不要完美** —— 写完初稿就发，允许后续修改
4. **中英文同步** —— 英文版本放在 `i18n/en/docusaurus-plugin-content-blog/` 同路径下

```bash
# 新建一篇文章
npm run new-post "标题" -- --slug post-slug
```

这条命令会自动生成中英文两个模板文件，省去手动建目录的麻烦。

## 关于数学公式

支持数学公式是我选 Docusaurus 的一个附加理由。大学时读信息论，傅里叶变换一直是最让我着迷的部分：

$$
F(\omega) = \int_{-\infty}^{\infty} f(t) e^{-i \omega t} \, dt
$$

时域和频域是看待世界的两种方式 —— 同一个信号，换个角度观察，往往会发现意想不到的规律。写博客也是类似：把脑子里的想法写出来，相当于从时域变换到频域，有些模糊的东西一下子就清晰了。

行内公式也很自然，比如 $E = mc^2$ 和 $x^2 + y^2 = z^2$。

## 一些零碎的想法

搭建这个博客的过程中，有些体会值得记下来：

- **减少摩擦** —— 写文章的步骤越多，写得越少。所以我把部署、评论、图片都一次性配好，只需要关心内容
- **保持简单** —— 没有花哨的动效，没有复杂的分类系统。文章、标签、归档，够了
- **数据在自己手里** —— 所有内容都是 Markdown 文件，随时可以迁移到任何平台

这些想法不只是关于写博客。做任何个人项目，减少摩擦、保持简单的原则都适用。
